# Forte AI Tender Intelligence

Forte AI Tender Intelligence — это система для интеллектуального анализа тендеров с использованием FastAPI, React, LLM-сервера и настраиваемой базы данных. Проект включает парсер, движок поиска, модуль категоризации с LLM, а также удобный интерфейс для анализа тендеров в реальном времени.

---

## Основной функционал

### 1. Работа с данными тендеров
Система поддерживает два формата хранения данных:

- CSV dataset (используется по умолчанию)
- MongoDB (включается через переменные окружения)

При запуске backend загружает данные, кэширует их в памяти и предоставляет быстрый интерфейс для поиска и фильтрации. Возможности:

- поиск по названию, описанию, ключевым словам
- фильтрация по категориям, суммам, периодам дат
- сортировка по сумме
- нормализация и унификация данных
- подготовка данных для LLM анализа

## Парсер & Данные
- goszakup_tenders_full_async.csv - это первоначальный спарсенный датасет
- tender_risk_model.pkl - это модель с комплаенсом и статистическим анализом

- service in jupyter.ipynb - это аналитический документ где я проводил анализ
- service_in_jupyter.py - это сервис
- ai_procure_parser.py - это разработанный парсер

---

### 2. LLM-сервер
Отдельный сервис обрабатывает запросы к языковой модели (OpenAI, Groq или другая совместимая API).

Функции LLM-сервера:

- категоризация тендеров
- анализ содержания и вывод инсайтов
- генерация описаний и объяснений
- определение ключевых признаков
- диалоговый режим для чатбота

Все маршруты доступны через REST API, интеграция с backend выполнена полностью.

---

### 3. Чатбот
Проект содержит рабочий чатбот с подключением к LLM-серверу. Интерфейс реализован в виде виджета на фронтенде.

Функциональность чатбота:

- кнопка-триггер (открытие/закрытие чата)
- состояние кнопки: «Чем я могу помочь?» / «Отмена»
- отправка текста на LLM-сервер
- отображение ответа с анимацией ожидания
- работа с контекстом и анализом тендера

Чатбот интегрирован в общий процесс поиска и анализа тендеров.

---

### 4. Frontend (React + Vite + TypeScript)
Frontend полностью переписан и оптимизирован. Использованы:

- Vite (быстрая сборка)
- React + TypeScript
- модульные компоненты для поиска и вывода результатов
- интерфейс чатбота
- стилизация через кастомные CSS-модули и общие стили

Основные компоненты:

- Topbar
- SearchRow
- Filters
- ResultsPanel
- ChatButton
- ChatWindow

Frontend взаимодействует с backend через REST API.

---

### 5. Backend (FastAPI)
Backend отвечает за:

- загрузку и кэширование CSV данных
- подключение к MongoDB при необходимости
- предоставление API для поиска, фильтрации и анализа
- проксирование запросов к LLM-серверу
- маршруты:
  - `POST /search`
  - `POST /analyze`
  - `POST /chat`
  - `GET /health`

Backend запускается через Docker и готов к продакшен-развёртыванию.

---

### 6. Парсер
В проект включён модуль парсинга тендеров. Основные возможности:

- сбор данных с внешних источников
- нормализация и очистка данных
- сохранение в CSV или MongoDB
- обновление локального датасета для последующего анализа

## Запуск проекта

### 1. Настроить переменные окружения

Создать файлы `.env` в `backend` и `frontend`:

Пример backend `.env`:

USE_MONGO=false
MONGO_URL=mongodb://localhost:27017
LLM_API_KEY=your_api_key


---

### 2. Запустить проект

docker compose up --build


Frontend будет доступен на http://localhost:5173  
Backend — на http://localhost:8000
